## 2025-02-14 - Fix DOM XSS in VaultView URL Launch
**Vulnerability:** A DOM-based Cross-Site Scripting (XSS) vulnerability existed in `app/components/vault/components/vault-view.js` where user-provided URLs were directly passed to `window.open` without protocol validation, allowing arbitrary JavaScript execution via `javascript:` URIs.
**Learning:** Security vulnerabilities often hide in unexpected inputs. `window.open` inherently trusts the provided scheme. Without explicit validation of the URL scheme to restrict to `http:` and `https:`, the browser happily executes `javascript:` URIs.
**Prevention:** Always parse untrusted URLs using the native `URL` constructor, and explicitly validate the `protocol` property against an allowlist of safe schemes (e.g., `['http:', 'https:']`) before using them in navigation or window opening contexts. Defensive attributes like `noopener,noreferrer` should also be added. Tests verifying the safety of these sinks should be written using generative techniques (like Property-Based Testing) to explore the input space aggressively.
