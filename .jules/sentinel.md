## 2024-05-18 - XSS in ModalManager via innerHTML String Interpolation
**Vulnerability:** Cross-Site Scripting (XSS) vulnerability existed in `ModalManager`'s UI rendering methods (`prompt`, `confirm`, `showForm`, `showAlert`). These methods dynamically generated HTML structures by directly interpolating unescaped user inputs (`title`, `message`, `label`, `type`) into `.innerHTML` assignments.
**Learning:** Bypassing DOM text assignment logic (like `.textContent`) by directly using string interpolation for `.innerHTML` leaves the application extremely vulnerable to XSS. Unescaped inputs can break out of elements and inject malicious executable HTML, especially via attribute breakouts (`" onerror=alert(1)`). An absence of robust sanitization exposes components dealing with direct UI rendering to dangerous payload execution.
**Prevention:** To prevent XSS when using `innerHTML` in UI components like `ModalManager`, implement a robust HTML escaping method (e.g., `#escape(str)`) that safely replaces special characters (`&`, `<`, `>`, `"`, `'`) with their respective HTML entities. All dynamic inputs must be explicitly sanitized before interpolation. Validate the fix with generative property-based testing (PBT) leveraging adversarial input arrays containing boundary breakouts and malicious payloads.
