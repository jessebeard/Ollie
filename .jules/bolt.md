## 2024-05-18 - Math.max/min clamping in Uint8ClampedArray
**Learning:** When calculating and assigning pixel values directly to a `Uint8ClampedArray` (e.g., during image data assembly), avoid redundant manual bounds clamping and rounding using `Math.max(0, Math.min(255, Math.round(x)))`. The typed array natively handles [0, 255] clamping. However, DO NOT apply this optimization to plain objects or `Float32Array`s (like in `decoder.js` IDCT or `rgb-to-ycbcr.js`), as they lack native clamping, and removing explicit bounds checking will cause core JPEG math regressions.
**Action:** Remove redundant `Math.max(0, Math.min(255, Math.round(x)))` ONLY when assigning directly to a `Uint8ClampedArray`. Retain explicit clamping for all other numerical arrays and objects to preserve mathematical correctness.
