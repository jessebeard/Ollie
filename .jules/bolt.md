## 2024-03-24 - V8 Automatically Optimizes Loop-Invariant Code Motion\n**Learning:** Extracting simple multiplications out of inner loops in JavaScript (like `bytePos * blockCount`) does not produce measurable performance wins because modern engines like V8 naturally hoist these invariants. The optimization added complexity without benefit.\n**Action:** Avoid manual loop-invariant code motion for basic arithmetic in tight loops unless profiling specifically pinpoints it as a bottleneck in environments without advanced JITs.
## 2024-03-24 - Pre-allocate arrays instead of Array.prototype.push() in tight loops\n**Learning:** When flattening or combining large matrices in tight loops, repeatedly calling `Array.prototype.push()` incurs significant overhead from dynamic memory reallocations. By calculating the total size required upfront and initializing the array (`new Array(size)`), we can assign directly via an index tracker.\n**Action:** Use pre-allocated arrays where output sizes can be deterministically calculated ahead of time for heavy image data arrays.
