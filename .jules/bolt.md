## 2024-04-15 - Optimizing Entropy Loops and Stream Bitwriting
**Learning:** In hot entropy loops (`computeCategory` and `getBitRepresentation`), replacing the O(log N) bitwise `while` loop with `32 - Math.clz32(val)` transformed bit-length calculation into an O(1) operation using hardware instructions, gaining ~4x loop performance. Similarly, writing bits individually in a loop via `this.writeBit(bit)` is highly inefficient due to function overhead. Batching these into shift/mask operations across bytes in `BitWriter.writeBits` speeds up stream encoding significantly (~6x faster for bulk writes).
**Action:** When calculating bits for categorization in hot paths, leverage Math.clz32 along with exploiting known properties (e.g. knowing a number is already negative). When writing continuous bits to a stream, use bitwise masking and shifting to construct whole bytes rather than calling individual single-bit push functions.
