## 2024-04-25 - Hardware-accelerated bit length computation
**Learning:** In JavaScript, computing the bit-length of a number (e.g., determining a Huffman coding category or representing a bit sequence) using an O(log N) bitwise `while` loop is relatively slow due to loop overhead and interpreter branches. Using the built-in `Math.clz32(val)` transforms this into an O(1) hardware-accelerated instruction. When combined with avoiding `Math.abs()` on numbers known to be negative (e.g., computing `32 - Math.clz32(-val)` after a `val < 0` check), this provides a massive performance boost (~15x speedup) in hot encoding paths.
**Action:** Always prefer `32 - Math.clz32()` for computing bit length or most significant bit position over manual shifting loops, especially in hot paths like entropy coding.
